Writing about Emacs

At present I am using Emacs to produce a website. Here, I document how I am using Emacs in tandem with git to publish a website.

About the process: Org Mode

Org Mode is an Emacs package which comes built-in. I use Org Mode for writing webpages because I can simply export my notes to HTML. Org Mode manifests as a major mode of Emacs which is active when visiting .org files and special Org Mode buffers. Org is useful for managing projects by way of the TODO word. TODO items can be scheduled to specific times and added to an agenda. Org documents can be linked to one another using hyperlinks. It is possible to publish projects to HTML. This makes it simple to view org documents in the form of websites, complete with images. One can add custom HTML to the publishing function using code written in Emacs Lisp. I have used this to link to an external stylesheet and implement the navigation bar more consistently.

Git

Git is version control software which means it keeps track of changes to repositories of documents. It has network capabilities which allow one to clone repositories of software from remote servers. Alternatively, one can push changes to a remote location. I am using git to keep track of the development of my website. I am using GitHub Pages to host the website.
